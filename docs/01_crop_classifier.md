# Crop-Type Leaf Classifier — EfficientNet-B2

**Script:** `train_classifier.py`  
**Output directory:** `classifier_output/`  
**Role:** Stage 1 of the two-stage inference pipeline

---

## 1. Motivation

The YOLO26 and Faster RCNN detectors were trained on three crop types — Corn, Pepper, and Tomato — and include hard-negative OOD images to suppress detections on completely unrelated scenes. However, they were not trained to distinguish between *similar* leaf types outside the dataset. In practice, feeding a mango or banana leaf to the detector can cause it to return high-confidence disease detections (e.g., mango → "Tomato Late Blight") because the visual texture of healthy leaves is similar across species.

The crop-type classifier addresses this by acting as a **gating stage**:

1. Classify the input image as Corn, Pepper, Tomato, or Unknown.
2. If the crop type is Unknown (confidence below threshold), reject the image and skip detection entirely.
3. If a known crop type is identified, forward the image to the disease detector and filter its outputs to only the disease classes belonging to that crop.

This eliminates cross-crop false positives without modifying or retraining the underlying detector.

## 2. Architecture

| Component | Detail |
|---|---|
| Backbone | EfficientNet-B2 (pretrained on ImageNet-1K) |
| Input resolution | 260 × 260 px (EfficientNet-B2 native size) |
| Classifier head | `Linear(in_features=1408, out_features=3)` replaces default head |
| Output classes | 3 — Corn (0), Pepper (1), Tomato (2) |
| Parameters (total) | ~9.1 M |
| Parameters (trainable) | ~9.1 M (full fine-tuning) |
| Activation | Softmax at inference; raw logits during training |

EfficientNet-B2 was chosen over B0 or ResNet-50 for this task because:
- Its compound-scaled depth and width offer better feature discrimination than B0 for visually similar leaf textures.
- At 260 × 260 px input, it is substantially faster at inference than a 640 × 640 detector.
- Pre-trained ImageNet weights transfer well to agricultural leaf images — convergence is typically achieved within 10–15 epochs.

---

## 3. Training Data

The classifier is trained directly from the YOLO detection dataset using the existing train/valid/test splits. Class labels are derived automatically from the YOLO disease class IDs:

| YOLO class IDs | Crop | Classifier label |
|---|---|---|
| 0–4 | Corn | 0 |
| 5–14 | Pepper | 1 |
| 15–22 | Tomato | 2 |

The CSV files are generated by `generate_classifier_csv.py`:

```bash
python generate_classifier_csv.py
```

This produces:

| File | Images |
|---|---|
| `dataset/classifier_train.csv` | 2,861 |
| `dataset/classifier_valid.csv` | 1,024 |
| `dataset/classifier_test.csv` | 1,016 |

Images whose YOLO label files are missing or empty (hard-negative OOD images) are excluded — they carry no crop-type signal and would add noise to classification training.

**Class distribution (train split):**

| Crop | Count | Share |
|---|---|---|
| Corn | 817 | 28.6% |
| Pepper | 1,348 | 47.1% |
| Tomato | 696 | 24.3% |

The Pepper class is approximately 2× larger than the Tomato class. Label smoothing (`ε = 0.1`) in the cross-entropy loss mitigates this imbalance without requiring manual oversampling.

## 4. Training Configuration

| Hyperparameter | Value | Rationale |
|---|---|---|
| Epochs | 40 | Early stopping fires well before this in practice (~12–18 epochs) |
| Batch size | 64 | Optimal for M4 Pro 24 GB; scale up to 128 on A100 |
| Optimizer | AdamW | Superior to SGD for fine-tuning pre-trained models |
| Learning rate | 1e-4 | Standard for EfficientNet fine-tuning |
| Weight decay | 1e-4 | Regularises the full network including backbone |
| LR schedule | Linear warm-up (5 ep) → Cosine annealing to 1e-6 | Warm-up prevents destabilising pre-trained weights early |
| Loss function | Cross-entropy with label smoothing ε=0.1 | Handles class imbalance; reduces overconfidence |
| Gradient clipping | norm ≤ 1.0 | Stabilises fine-tuning, especially during warm-up |
| Early stopping | patience = 8 epochs | Halts after 8 consecutive non-improving validation epochs |
| Unknown threshold | 0.55 | Softmax confidence below this → reported as "unknown" at inference |

### Data augmentation (train split only)

| Transform | Parameters |
|---|---|
| Resize | 292 × 292 (pad before crop) |
| RandomCrop | 260 × 260 |
| RandomHorizontalFlip | p = 0.5 |
| RandomVerticalFlip | p = 0.5 |
| ColorJitter | brightness=0.3, contrast=0.3, saturation=0.3, hue=0.05 |
| RandomRotation | ±15° |
| Normalize | mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] |

## 5. Device & Mixed-Precision Strategy

| Device | AMP | GradScaler | Notes |
|---|---|---|---|
| CUDA | `torch.autocast(cuda, fp16)` | Yes — prevents fp16 underflow | `cudnn.benchmark=True` enabled; up to 8 DataLoader workers |
| MPS (Apple Silicon) | `torch.autocast(mps, fp16)` | No — MPS fp16 is numerically stable without scaling | 4 workers via `spawn` (Python ≥ 3.8 default on macOS) |
| CPU | None — full float32 | No | No overhead from autocast on CPU |

`pin_memory` is enabled only on CUDA. On MPS, CPU and GPU share the same physical RAM (unified memory) so pinning adds overhead rather than removing it.

DataLoader workers are kept alive across all epochs via `persistent_workers=True`, eliminating the per-epoch `spawn` startup cost on macOS.

## 7. Output Files

After training, the following files are written to `classifier_output/`:

```text
classifier_output/
├── best.pth                    ← Best validation-accuracy checkpoint (CPU weights)
├── last.pth                    ← End-of-training checkpoint (CPU weights)
├── metrics_history.json        ← Per-epoch train/val loss and accuracy
└── figures/
    ├── fig_01_training_curves.png     ← Loss and accuracy curves (train vs val)
    ├── fig_02_confusion_matrix.png    ← Counts + normalised confusion matrix (test set)
    ├── fig_03_per_class_accuracy.png  ← Per-class accuracy bar chart (test set)
    └── fig_04_sample_predictions.png  ← 12 sample images with true/predicted labels
```

Checkpoint format:
```python
{
    "epoch":     int,         # epoch at which this checkpoint was saved
    "model":     state_dict,  # weights on CPU (portable across all devices)
    "valid_acc": float,       # validation accuracy at save time
}
```

Weights are always saved as CPU tensors so checkpoints load cleanly on any device — MPS, CUDA, or CPU — without requiring `map_location` remapping.

## 8. Inference Integration

The `CropClassifier` class at the bottom of `train_classifier.py` is importable directly:

```python
from train_classifier import CropClassifier

clf = CropClassifier()                          # auto-loads classifier_output/best.pth
crop, confidence, yolo_ids = clf.predict("leaf.jpg")

# crop        → "Corn" | "Pepper" | "Tomato" | "unknown"
# confidence  → softmax probability of the top class (0.0–1.0)
# yolo_ids    → YOLO class IDs to pass to the detector filter
#               e.g. [0,1,2,3,4] for Corn; [] for unknown
```

Use in a two-stage detection pipeline:

```python
from train_classifier import CropClassifier
from ultralytics import YOLO

clf  = CropClassifier(confidence_threshold=0.55)
yolo = YOLO("runs/crop_disease_yolo26/weights/best.pt")

def detect(image_path: str):
    crop, conf, yolo_ids = clf.predict(image_path)
    if crop == "unknown":
        return {"crop": "unknown", "confidence": conf, "detections": []}

    results = yolo(image_path)[0]
    # Filter boxes to only the detected crop's disease classes
    filtered = [
        box for box in results.boxes
        if int(box.cls) in yolo_ids
    ]
    return {"crop": crop, "confidence": conf, "detections": filtered}
```

### Adjusting the confidence threshold

The default threshold of 0.55 was chosen to balance false rejections (valid crop images flagged as unknown) against false acceptances (non-crop images passed to the detector). For field deployment:

- **Lower threshold (e.g. 0.45):** fewer false rejections, but more non-crop images reach the detector.
- **Higher threshold (e.g. 0.70):** stricter gating; unknown crops are rejected more reliably, but more valid images may be rejected on blurry or partially-occluded shots.
