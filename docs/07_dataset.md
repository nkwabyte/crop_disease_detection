# Dataset Documentation

**Source:** Ghana Crop Disease Challenge v2  
**License:** CC BY 4.0  
**Publisher:** Roboflow Universe (ghanacropdiseasechallenge workspace)

---

## 1. Overview

The dataset contains annotated leaf images for three West African crop species:

- **Corn** (*Zea mays*) — 5 disease/health classes
- **Pepper** (*Capsicum* spp.) — 10 disease/health classes
- **Tomato** (*Solanum lycopersicum*) — 8 disease/health classes

**Total: 23 classes** across 5,239 images (after hard-negative exclusion).

---

## 2. Dataset Splits

### YOLO format (`data/main/`)

Images are stored as JPEG with corresponding YOLO `.txt` label files (one per image). Each label line: `class_id cx cy w h` (all normalised to [0, 1]).

| Split | Images | Bounding Boxes |
|---|---|---|
| Train | 3,199 | ~21,625 |
| Valid | 1,024 | ~7,100 |
| Test | 1,016 | ~7,050 |
| **Total** | **5,239** | **~35,775** |

### Faster RCNN / CSV format (`dataset/`)

Bounding boxes expressed as absolute pixel XYXY coordinates in CSV files.

| File | Rows (annotations) |
|---|---|
| `final_train_labels.csv` | ~21,625 |
| `final_validate_labels.csv` | ~7,100 |
| `final_test_labels.csv` | ~7,050 |

CSV columns:

| Column | Type | Description |
|---|---|---|
| `width` | int | Image width in pixels (640) |
| `height` | int | Image height in pixels (640) |
| `crop` | str | Crop type: Corn, Pepper, or Tomato |
| `class` | str | Full disease class name (e.g. "Tomato Late Blight") |
| `fname` | str | Image filename |
| `img_path` | str | Relative path to image file |
| `x1, y1, x2, y2` | float | Bounding box in absolute pixels (XYXY) |
| `id` | int | Row index |
| `integer_label` | int | 1-indexed class label (matches `label_map.json`) |

### Classifier format (`dataset/`)

Generated by `generate_classifier_csv.py` — one row per image (not per bounding box):

| File | Images |
|---|---|
| `classifier_train.csv` | 2,861 |
| `classifier_valid.csv` | 1,024 |
| `classifier_test.csv` | 1,016 |

CSV columns: `image_path, crop_label, crop_id, split`

Hard-negative images (which have no disease annotations) are excluded from the classifier CSVs.

### Label indexing convention

| Pipeline | Indexing | Background |
|---|---|---|
| YOLO | 0-indexed (0–22) | No background class — empty `.txt` = no objects |
| Faster RCNN | 1-indexed (1–23) | Class 0 = background (reserved by Faster RCNN) |
| Classifier | 0-indexed (Corn=0, Pepper=1, Tomato=2) | N/A — classification, not detection |

`dataset/label_map.json` uses 1-indexed values (Faster RCNN convention).

The dataset is **highly imbalanced**: `Tomato_Septoria` (~3,988) has nearly 200× more annotations than `Pepper_Early_Blight` (~21). This imbalance is not corrected by oversampling in either pipeline; instead:
- YOLO26 uses label smoothing (ε=0.10) and its built-in class-frequency weighting.
- Faster RCNN uses the RoI sampler which balances positive/negative proposals per image.
- The classifier uses label smoothing (ε=0.10) with AdamW.

## 5. Image Characteristics

| Property | Value |
|---|---|
| Resolution | 640 × 640 px |
| Format | JPEG |
| Colour space | RGB |
| Annotations per image | 1–12 bounding boxes |
| Median boxes per image | ~4 |

All images are 640 × 640 — no resizing is required before feeding to any of the models.

## 6. Hard Negatives

Hard-negative images are not part of the original Ghana Crop Disease Challenge dataset. They are downloaded by the training scripts from `picsum.photos` and stored in `data/negatives/`.

| Pipeline | Count | Source |
|---|---|---|
| YOLO26 | 300 | Random seeds 1–1999 (seed 42 shuffle) |
| Faster RCNN baseline | 200 | Same source |
| SE-FPN final | 300 | Categorised: 7 semantic groups (see below) |
| Classifier | 0 | Excluded from classifier CSVs |

SE-FPN categorised negatives:

| Category | Count |
|---|---|
| Animals | 50 |
| People | 50 |
| Cityscape | 40 |
| Landscape | 40 |
| Transport | 35 |
| Objects | 45 |
| Indoor | 40 |

All hard-negative images share the `data/negatives/` directory. Running all three training scripts will populate the folder with up to 300 images total (each script only downloads images it needs).

## 7. Data Access

The dataset is available at:

```
https://universe.roboflow.com/ghanacropdiseasechallenge/ghana-crop-disease-challenge/dataset/2
```

**Download (YOLO format):**

```bash
# Using Roboflow API
pip install roboflow
from roboflow import Roboflow
rf = Roboflow(api_key="YOUR_KEY")
project = rf.workspace("ghanacropdiseasechallenge").project("ghana-crop-disease-challenge")
dataset = project.version(2).download("yolov8")
```

Place the downloaded dataset at `data/main/`. The `README.dataset.txt` and `README.roboflow.txt` files inside `data/main/` contain the original Roboflow provenance information.
